## Claude Code vs GitHub Copilot vs 纯人工：C++ 开发效率对照实验

> 论文配套材料 — P2-2 对照实验
> 作者：赵鑫
> 实验日期：2025年8月（待执行）

---

## 1. 研究问题与假设

### 研究问题

- RQ1：Claude Code 辅助开发的任务完成时间是否显著短于 GitHub Copilot 和纯人工？
- RQ2：Claude Code 生成的代码在首次编译通过率上是否高于 GitHub Copilot？
- RQ3：三种方式在代码测试覆盖率上是否存在显著差异？
- RQ4：三种方式在代码质量评分（人工评审）上是否存在显著差异？

### 假设

- H1：Claude Code < Copilot < 纯人工（任务完成时间）
- H2：Claude Code > Copilot（首次编译通过率）
- H3：Claude Code > Copilot > 纯人工（测试覆盖率，AI 更擅长生成测试）
- H4：纯人工 ≥ Claude Code > Copilot（代码质量，AI 生成代码在精细度上不及人类）

## 2. 实验设计

### 设计类型

3（工具组）× 5（任务）混合设计

### 自变量

- 工具类型（被试间因素）：
  - 组 A：Claude Code v1.0.43 + Claude Sonnet 4
  - 组 B：GitHub Copilot (VS Code 插件, 2025年7月版)
  - 组 C：纯人工（仅允许查阅 cppreference.com 和 Stack Overflow）

### 因变量（测量指标）

| 指标 | 缩写 | 测量方法 |
|------|------|---------|
| 任务完成时间 | TTC | 从接收任务到所有测试通过的时间（分钟），精确到秒 |
| 首次编译通过率 | FCP | 首次编译 0 error 的比例（0/1） |
| 编译错误修复轮次 | CEF | 从首次编译到无错误所需的编译次数 |
| 测试覆盖率 | COV | gcov/lcov 行覆盖率 (%) |
| 代码质量评分 | CQS | 3名评审独立打分（满分10分）的平均值 |
| 人工修改占比 | HMR | 人工修改行数 / 总行数 (%) (仅 AI 组) |

## 7. 分析计划

### 7.1 描述性统计

对每组在每个任务上计算：均值、标准差、中位数、最小值、最大值。

### 7.2 推断统计

```
RQ1 (TTC):   混合效应模型 — TTC ~ tool + (1|participant) + (1|task)
RQ2 (FCP):   Fisher's exact test (组间首次编译通过率比较)
RQ3 (COV):   Kruskal-Wallis test (覆盖率数据通常非正态)
RQ4 (CQS):   Intraclass Correlation Coefficient (评审者间信度) + Kruskal-Wallis
```

显著性水平：α = 0.05。多重比较校正：Holm-Bonferroni。

### 7.3 效应量

- Cohen's d (连续变量)
- Cramér's V (分类变量)
- 报告 95% 置信区间

---

## 8. 局限性声明

1. **样本量小**（每组4人）：统计功效有限，适合作为探索性/试点研究
2. **语言特异性**：仅测试 C++，结果不能推广到其他语言
3. **任务选择偏差**：5个任务不能代表所有C++开发场景
4. **工具版本效应**：AI工具迭代快速，结果的时间稳定性有限
5. **实验室环境**：控制条件下表现可能与真实工作环境不同

## 附录 A：任务规格说明

### T1: SPSC 环形缓冲区

```
实现一个线程安全的单生产者-单消费者（SPSC）环形缓冲区。

要求：
1. 使用 C++17 标准
2. 容量必须是2的幂（用于位运算优化取模）
3. 支持单元素 push()/pop() 和批量 push_batch()/pop_batch()
4. 使用 std::atomic<size_t> 作为 head/tail 索引
5. 使用 alignas(64) 防止伪共享
6. 编写 Google Test 单元测试（至少3个测试用例）

验收标准：
- 所有测试通过
- 使用 clang-tidy 无严重警告
- 代码覆盖率 ≥ 80%
```

### T2: 异步 HTTP GET 请求

```
实现一个函数：使用 C++ 标准库或 Boost.Asio，异步发起 HTTP GET 请求

要求：
1. 函数签名：std::future<std::string> async_http_get(const std::string& url)
2. 支持超时参数（默认5秒）
3. 返回响应体（body）作为 std::string
4. 编写 Google Test 测试（使用 httpbin.org 或 mock）
5. 处理以下错误情况：连接失败、超时、非200状态码

验收标准：
- Mock 测试通过（不需要真实网络）
- 超时逻辑正确
```
